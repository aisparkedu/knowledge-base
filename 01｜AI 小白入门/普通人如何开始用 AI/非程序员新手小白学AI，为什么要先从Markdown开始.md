非程序员新手小白学AI，为什么要先从Markdown开始

很多人一上来学 AI，先去研究大模型、智能体、工作流、自动化工具。

但我越来越觉得，如果你真想把 AI 用起来，第一步反而不用那么急着追新词。

先把 Markdown 学会，后面你学 AI 会轻松很多。

因为很多人卡住，并不是不会问 AI，也不是不会装软件，而是不会整理内容、不会沉淀资料、不会把自己的想法变成一个清晰的结构。

而 Markdown，恰好就是那个最简单、最轻量、最适合普通人入门的工具。

**一、为什么学 AI 要先学 Markdown**

很多人对 Markdown 的第一反应是：这不就是写点符号吗？有什么好学的？

但你真正用久了就会发现，它不是“写符号”，而是在训练你把内容写得更清楚、更结构化、更方便复用。

你写一篇笔记、整理一次提示词、记录一个工作流、沉淀一个教程，本质上都在做同一件事：把脑子里的东西变成可以复用的文本结构。

而 AI 最喜欢处理的，恰恰就是这种结构清晰的内容。

如果你的资料本来就乱糟糟的，今天放微信群，明天放备忘录，后天又丢进某个聊天窗口，那后面想让 AI 帮你总结、改写、生成、检索，效率一定会很低。

Markdown 的价值就在这里：**它不是一门复杂技术，但它是你搭建 AI 工作流之前，最值得先打好的底层习惯。**

**二、为什么我更推荐用 Obsidian**

如果你只是临时记两句内容，任何笔记软件都行。

但如果你后面真的想把 AI 用进自己的学习、写作和自媒体工作流里，我更推荐你早点开始用 `Obsidian`。

原因不是它有多高级，而是它很适合长期积累。

第一，它的内容本质上就是本地 Markdown 文件。

这意味着你写下来的每一篇笔记、每一个选题、每一份提示词，都是标准化的文本。后面无论你要拿去喂给 AI，总结成知识库，还是接入自动化工具，都更方便。

第二，它天然适合做连接。

你今天记一篇“AI 入门词汇”，明天记一篇“公众号选题”，后天再记一篇“提示词模板”，这些内容不是一篇篇孤立的文档，而是可以慢慢串成你自己的资料库。

第三，它对后续工作流非常友好。

你以后如果要做这些事：

- 用 AI 改写公众号
- 搭建自己的内容知识库
- 做提示词仓库
- 整理选题库和素材库
- 接自动化脚本和工作流工具

你会发现，`Obsidian + Markdown` 这个组合特别省事。

因为前面内容已经是干净的文本格式了，后面接 AI、接脚本、接自动化时，阻力会小很多。

说得直接一点，很多人后面工作流搭不起来，不是不会用 AI，而是前面的内容管理太乱了。

**三、Markdown 到底适合拿来做什么**

如果你是非程序员，其实完全不用把 Markdown 想得太技术。

你可以把它理解成一种“更适合长期整理内容的写作方式”。

它最适合做的事情包括：

- 写学习笔记
- 记录 AI 工具使用方法
- 整理提示词模板
- 写公众号草稿
- 做课程笔记
- 沉淀自己的知识卡片

尤其是写公众号这件事，Markdown 真的很好用。

因为你写的时候更专注内容本身，不会一直被字体、字号、颜色这些排版细节打断。先把内容结构写清楚，后面再做公众号排版，效率会高很多。

**四、最常见的 Markdown 指令**

如果你从来没接触过 Markdown，也不用紧张。

真的常用的，其实就那么几个，学会以后基本就能写大部分内容了。

1. 标题

```Markdown
# 一级标题## 二级标题### 三级标题
```

`#` 越多，标题层级越低。

1. 加粗

```Markdown
**这是加粗内容**
```

适合强调重点词和关键句。

1. 斜体

```Markdown
*这是斜体内容*
```

斜体没有加粗常用，但有时做轻强调会很方便。

1. 列表

```Markdown
- 第一项
- 第二项
- 第三项
```

如果你想写步骤、清单、方法总结，列表会特别常用。

1. 有序列表

```Markdown
1. 第一步
2. 第二步
3. 第三步
```

适合教程、流程说明这类内容。

1. 引用

```Markdown
> 这是一段引用内容
```

常用来放提醒、原话、总结句。

1. 插入链接

```Markdown
[OpenAI](https://openai.com)
```

以后整理资料和工具网站时会很常见。

1. 插入图片

```Markdown
![图片说明](图片地址)
```

如果你后面做图文笔记，这个也会用到。

1. 行内代码

```Markdown
`Obsidian`
```

适合标记软件名、命令、提示词变量这类内容。

1. 代码块

````Markdown
```python
print("hello")
````

````Plain Text

以后如果你要记录脚本、提示词模板、配置内容，代码块会特别清晰。

**11. 分隔线**

```md
---
````

适合把一篇长笔记拆成不同部分。

**常见误区：** 不要一开始就想把所有语法背下来。先学最常用的标题、加粗、列表、引用，就已经够你写绝大多数笔记了。

**五、为什么这一步会影响你后面的 AI 工作流**

很多人以为 AI 工作流的重点是选模型、选工具、选插件。

但真正用久了你会发现，工作流能不能搭起来，前提往往是你有没有一套稳定的内容输入方式。

Markdown 让你能稳定写内容，Obsidian 让你能稳定存内容。

等你后面开始做这些事的时候：

- 让 AI 帮你总结笔记
- 让 AI 改写成公众号
- 把零散想法整理成选题
- 把一堆资料做成知识库
- 把写作流程接进自动化工具

你会很明显地感受到，前面学会 Markdown 这件事，真的不是绕路，而是在铺路。

**最后想说**

如果你现在刚开始学 AI，又总觉得各种工具太多、概念太杂，那不如先别急着追最前沿。

先把 `Markdown` 学会，再把 `Obsidian` 用起来。

你会发现，这一步看起来最基础，反而最适合普通人建立自己的 AI 使用习惯。

等你后面真的开始搭内容工作流、知识库工作流、写作工作流时，你会感谢自己今天先把这块打稳了。
