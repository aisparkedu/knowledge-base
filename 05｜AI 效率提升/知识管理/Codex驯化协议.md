# Codex 驯化协议（Codex Taming Protocol）

> 让 AI 越用越懂你的长期记忆工作区。
>
> 这份协议可直接发给观众，已去个人化、不含私人信息，可整份转发。
> 用法：先用【Part 1 一键安装指令词】让 Codex 自动建好工作区；再用【Part 2 核心指令词】日常驱动；
> 不想用 AI 自动建的，照【Part 3 文档模板原文】手动复制也行。

---

## Part 1 · 一键安装指令词（最省事，复制这段丢给 Codex / Cursor）

> 打开 Codex（或 Cursor 的 Agent），把下面整段发给它。它会自动建好整套工作区。

```
帮我建一个长期记忆工作区，让你越用越懂我。请在我的 ~/Workspace 目录下创建下面这套文件
（没有这个目录就先建），每个文件按我标注的作用写好初始内容，涉及我个人信息的地方先留占位等我补：

1. AGENTS.md —— 你的全局行为规则，每次新对话第一件事就读它。要求写明：
   ① 开工先读 00_全局控制台.md 和 01_全局复盘与踩坑日志.md；
   ② 所有项目只在本 Workspace 内进行，禁止扫描或修改 Workspace 以外的文件（我明确要求才可以）；
   ③ 我的原始素材只读、不准覆盖；
   ④ 复盘义务：只要我让你返工或纠正做法，任务结束后你必须主动把「坑+正确做法」追加进 01，不用我提醒；
   ⑤ 统一用简体中文交流。
2. 00_全局控制台.md —— 总入口，含三块：「我是谁」「工作区规矩索引」「进行中的项目表」。
3. 04_个人信息档案.md —— 记录我的身份、设备、常用工具、工作偏好，先用【】占位等我填。
4. 01_全局复盘与踩坑日志.md —— 错题本，规则是「你自己记、不用我提醒」，先留好格式和一条示例。
5. 02_工作区命名与架构规范.md —— 目录和命名规矩：一个项目一个文件夹、交付文件带版本号、禁止散落到桌面/下载。
6. 03_新项目创建SOP.md —— 我说「开个新项目」时你的固定建项流程。

建好后列一下目录结构给我看，并提醒我去 04_个人信息档案.md 把个人信息补上。
从今往后每次开新对话，你都要先读 00 和 01 再干活。
```

> 建完记得做一件事：打开 `04_个人信息档案.md` 把你的真实信息填进去（身份、电脑、常用工具），
> 这是让 Codex「认识你」的关键，填得越具体，它越不会给你通用废话。

---

## Part 2 · 核心指令词（工作区建好后，日常这样驱动它）

**① 让它认识你、列出最该交给它的 10 件事（视频第一步同款）**
```
读一下我的「个人信息档案」，再快速看一眼 ~/Workspace 里的项目，
结合这些列出最该交给你的 10 件提效的事，每条一句话说清能帮我省什么。
不用扫描整台电脑。
```

**② 让它读踩坑日志、主动避坑（视频里的"实锤"同款）**
```
读一下我工作区里的「全局复盘与踩坑日志」，告诉我：下次做同类任务时，
你会主动避开哪 3 个坑？请直接对应日志里的条目回答。
```

**③ 开新项目（让它按规矩建，不再到处乱建文件）**
```
我要开个新项目：【一句话说要做什么】。
按 03_新项目创建SOP.md 的流程在 ~/Workspace 里建好项目文件夹和 README，
并在 00_全局控制台.md 的项目表里登记一行。
```

> 进阶：每次任务结束，你只要说一句「把这次的坑记进日志」，它就会自己往 01 里追加错题，
> 时间越久，这个错题本越值钱——这才是「越用越聪明」的真正来源。

---

## Part 3 · 文档模板原文（想手动建 / 想检查 AI 建得对不对，照这个）

### 文件 1：AGENTS.md
```markdown
# AGENTS.md — 全局工作规则

## 启动必做
1. 每次新对话开始，先读 `00_全局控制台.md` 了解全局状态。
2. 接到任务前，先读 `01_全局复盘与踩坑日志.md`，确认不重犯已记录的错误。

## 工作边界
1. 所有项目必须在本 Workspace 文件夹内进行。
2. 禁止扫描或修改 Workspace 以外的文件，除非我明确要求。
3. 我提供的原始素材只读，产出写到项目的 `产出/` 目录，禁止覆盖原件。

## 命名与结构
1. 严格遵守 `02_工作区命名与架构规范.md`。
2. 新项目按 `03_新项目创建SOP.md` 流程创建。

## 复盘义务（最重要）
1. 只要我表达不满、要求返工或纠正做法，任务结束后必须主动把「坑 + 正确做法」
   追加到 `01_全局复盘与踩坑日志.md`，不需要我提醒。
2. 每完成一个项目阶段，主动更新项目 README 的进度和下一步。

## 工作约定
1. 始终用简体中文交流。
2. 输出新版本用新文件名，不覆盖旧文件。
```

### 文件 5：02_工作区命名与架构规范.md
````markdown
# 工作区命名与架构规范

> 这是 AI 的行为说明书，所有任务必须遵守。

## 目录规矩
1. 所有项目必须在本 Workspace 文件夹内进行，禁止在桌面、下载、根目录等位置创建工作文件。
2. 一个项目一个文件夹，禁止把多个不相关任务混在同一个文件夹。
3. 临时文件放项目内的 `_tmp/`，任务结束由 AI 主动清理。

## 命名规矩
1. 项目文件夹：`Project_编号_项目名`，编号两位数递增，例如 `Project_01_xxx`。
2. 项目名用短语，看名字就知道干什么。
3. 交付文件带版本：`方案_v1.md`、`成片-v2.mp4`，禁止用「最终版」「新建文档」这类名字。
4. 顶层全局文档用 `00_ / 01_ / 02_ / 03_` 编号前缀，永远排在项目列表前面。

## 项目内部结构
```
Project_XX_项目名/
├── README.md   ← 项目目标、当前进度、下一步（AI 维护）
├── 素材/        ← 输入的原始材料（只读，禁止修改原件）
├── 产出/        ← 交付物
└── _tmp/        ← 临时文件，可随时删除
```

## 工作流程规矩
1. 开新项目必须走 03_新项目创建SOP.md 的流程。
2. 每次任务开始前，先读 01_全局复盘与踩坑日志.md。
3. 每次任务结束后，更新项目 README 进度；有返工则追加踩坑日志。
4. 禁止扫描 Workspace 以外的目录，除非我明确要求。
````

### 文件 6：03_新项目创建SOP.md
````markdown
# 新项目创建 SOP

> 每次我说「开个新项目 / 帮我做 XX」且不属于现有项目时，AI 按此流程执行。

## 流程
1. 确认归属：先判断属于现有项目还是新项目，拿不准就问我一句。
2. 取编号：查看当前最大的 Project 编号，递增 1。
3. 建结构：按规范创建文件夹和子目录（README / 素材 / 产出 / _tmp）。
4. 写 README：项目目标（一句话）、当前进度、下一步。
5. 登记：在 00_全局控制台.md 的项目表中追加一行。
6. 开工前自查：读一遍 01_全局复盘与踩坑日志.md，确认本次不会重犯旧错。
````
